# 语音训练与联动方案阶段报告

## 截止时间
- 2026-04-10

## 当前阶段
- `AstrBot 联动`：`Phase 3` 已完成，复杂任务协处理器已具备事件筛选、群级分路由和运行时热更新
- `语音训练`：`Phase 5` 已完成，训练中心已具备浏览器质检修订、大文件分片上传、发布/回滚与版本闭环

## 已完成

### AstrBot 联动
- 建立“复杂任务委托”规则引擎
- 保留 `/Astrbot` 显式委托
- 自动委托仅作用于复杂任务，不接管日常聊天主链路
- 失败自动回退到 QQTalker 本地处理
- 支持群级 allowlist
- 支持群级 denylist，且 denylist 优先于 allowlist
- 支持群级 route override：`local-only / force-delegate`
- 支持运行时热更新复杂任务委托配置
- 控制台已接入复杂任务配置与状态摘要
- 控制台已接入联动详情面板，可查看决策计数、最近事件、最近决策与事件筛选

### 语音训练
- 新增训练工作区服务
- 新增训练中心总览 API
- 新增训练详情 API
- 新增训练轻任务执行 API
- 新增训练任务状态 API
- 新增本地素材路径导入到 `training/raw`
- 新增浏览器单文件上传到 `training/raw`
- 新增浏览器大文件分片上传到 `training/raw`
- 新增最近任务历史持久化
- 新增后台任务队列与前端轮询状态流
- 新增训练结果自动发布到 `data/voice-models`
- 新增 `voice-model.json / catalog.json` 自动回写
- 新增发布历史、快照备份与回滚能力
- 新增浏览器内数据质检与转写修订能力
- 控制台新增“语音训练中心（MVP）”
- 可查看每个角色的来源数、版本数、切片数、可用样本数和就绪状态
- 可查看角色训练详情
- 可触发轻任务：
  - `sync`
  - `clips-suggest`
  - `transcribe`
  - `manifest`
  - `eval`
  - `import-raw`
  - `upload-raw`
  - `publish-model`
  - `rollback-model`
  - `review`
  - `upload-chunk`

## 未完成
- 当前这份“语音训练与联动方案”里的既定主线能力已全部落地
- 若继续演进，更偏向增强项而非方案内遗留项：
  - 训练条目波形试听、逐条音频预览与更细的质检辅助
  - 更长周期的 AstrBot 事件持久化与历史检索
  - 更细粒度的训练版本审批策略

## 当前结论
- 这份“语音训练与联动方案”现在可以视为已整体完成。
- 当前语音训练部分可视为 `Phase 5`：已经具备工作区控制面、轻任务入口、本地路径导入、浏览器单文件/大文件上传、浏览器内质检修订、后台队列和状态流、训练结果入库、发布历史与回滚闭环。
- 当前 `AstrBot` 联动部分可视为 `Phase 3`：在“QQTalker 主导、AstrBot 只处理复杂任务”的架构原则下，已经具备 allowlist / denylist / route override、事件筛选、运行时热更新与失败回退能力。
